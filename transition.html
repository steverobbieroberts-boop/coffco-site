1. Introduction

This plan defines how the CoffCo in-room hotel coffee machine is transitioned from development into operational service at Vacation Stables. Two stages are addressed: Preparing for Transition (TNA and TEPIDOIL analysis) and Performing Transition (commissioning). Two customers are served: Vacation Stables as the operational receiver and the hotel room guest as the end user.

2. Technical Publications — Delivery Strategy

2.1 Online-First Approach

All CoffCo technical publications are delivered digitally. The CoffCo machine is a connected IoT product with a cloud operations platform and OTA firmware update capability. Delivering manuals on paper creates a version control problem that conflicts directly with the system architecture: when firmware changes, paper manuals become incorrect. Digital delivery solves this at the source — manuals are updated centrally and all users access the current version automatically.

A secondary driver is the hotel room environment. Hotel guests do not read paper manuals — SRS 2.1 mandates that the machine must be operable without any documentation at all. Housekeeping staff work across multiple rooms and cannot carry physical binders. Field engineers work in plant rooms and hotel corridors where paper gets damaged. Digital delivery is therefore both the practical and the principled choice.

2.3 Version Control

All online manuals are version-controlled on the CoffCo content management platform. A version number and effective date appear in the header of every page. When a new firmware version changes an operating procedure, the relevant manual section is updated before the firmware update is deployed. Field engineers and property managers are notified of manual updates via the cloud ops dashboard. The offline PDF on the Vacation Stables maintenance tablet is replaced at the next field engineer visit.

4. Maintenance Manual — FIDA Format

4.1 Why FIDA

The CoffCo Maintenance Manual is structured in FIDA (Fault Identification, Diagnosis, Action) format. FIDA is the appropriate format for a technically-capable maintainer working on a system they did not design. It provides a structured, systematic path from observable symptom to correct resolution action, without requiring the maintainer to understand the full system architecture.

FIDA is particularly well-suited to the CoffCo assembly-swap maintenance model. The Vacation Stables maintenance team and CoffCo field engineers do not repair failed components — they identify the faulty assembly, verify the diagnosis, and swap it. FIDA maps directly to this three-step process: Identify what is wrong → Diagnose which assembly → Act to resolve. It also determines the correct escalation point: if the action cannot be completed at the maintenance team level, FIDA tells the maintainer to escalate to the CoffCo field engineer before attempting anything further.

4.2 FIDA Structure

StepFIDA ElementWhat the Maintainer Does
1Fault IdentificationIdentify the observable symptom — either from the machine status indicator (FAULT / MAINTENANCE REQUIRED state), from a cloud ops alert (MILK-SPOILAGE-RISK, COMMS-FAULT, SAFE-SHUTDOWN), from a guest report, or from a fault code on the CoffCo diagnostic tool. This step produces a Fault ID — a specific symptom label that is the entry point to the Diagnosis section.
2DiagnosisCarry out the structured checks and tests specified for the identified fault. Each diagnosis procedure isolates the fault to one assembly (Control Assembly, Thermal Assembly, PSU, Fluid Store, or Structural Chassis) or determines that the fault requires field engineer escalation. Diagnosis uses: visual inspection, status indicator state, cloud alert type, and where applicable, the CoffCo diagnostic tool (field engineers only).
3ActionCarry out the specified corrective action. For Vacation Stables maintenance team: this is limited to power cycle, consumable clear, drip tray service, or escalation to CoffCo field engineer. For CoffCo field engineer: this includes assembly swap, firmware update, safety interlock test, and HACCP fluid path cleaning after swap.

4.4 FIDA Worked Examples

The following three examples show how the FIDA structure is applied to common CoffCo fault scenarios. These are representative entries from the Maintenance Manual.

FIDA Example 1 — Machine shows MAINTENANCE REQUIRED
F — Fault IdentificationSymptom / Entry Point
Observable symptomStatus indicator shows MAINTENANCE REQUIRED (amber). Cloud ops dashboard shows alert. Guest has reported 'machine won't start'.
D — DiagnosisStructured Checks
Check sequenceD1: Is the drip tray full? — Remove drip tray and check. If full → go to Action A1. D2: Is water reservoir empty? — Check reservoir level. If empty → go to Action A2. D3: Is consumable supply low or empty? — Check consumable count. If low → go to Action A3. D4: Is COMMS-FAULT also shown? — Check cloud dashboard. If COMMS-FAULT → go to Action A4. D5: None of the above — go to Action A5 (escalate).
A — ActionResolution
Actions by diagnosis outcomeA1 (drip tray full): Empty and clean drip tray. Refit. Confirm MAINTENANCE REQUIRED clears. [VS maintenance team] A2 (empty reservoir): Refill water reservoir to MAX. Confirm status clears. [VS maintenance team] A3 (low consumable): Restock consumables from floor store. [Housekeeping / VS maintenance team] A4 (COMMS-FAULT): Check hotel Wi-Fi status. Power cycle machine. If alert persists >60 min — log with CoffCo cloud ops. [VS maintenance team] A5 (unresolved): Escalate to CoffCo field engineer. Record fault code if available.
FIDA Example 2 — Machine enters SAFE-SHUTDOWN [SAFETY-CRITICAL — escalate immediately]
F — Fault IdentificationSymptom / Entry Point
Observable symptomMachine is unresponsive. Status indicator shows FAULT (red, steady). Cloud ops alert SAFE-SHUTDOWN received. Machine does not respond to controls. No brew possible.
D — DiagnosisStructured Checks
Check sequenceD1: Has the machine been physically disturbed or tipped? — Visual check. If yes, check for fluid spillage before any further action. D2: Is there any smell of burning or visible smoke? — If yes: do not touch. Evacuate room and call hotel emergency procedure immediately. D3: Is there water or liquid around the base of the machine? — If yes: do not touch. Isolate mains at room socket. Call CoffCo field engineer immediately. D4: None of the above — power cycle machine (switch off at mains socket for 60 seconds, then restore). Does SAFE-SHUTDOWN clear? → Yes (clears to STANDBY): Monitor for 30 minutes. If re-enters SAFE-SHUTDOWN, go to D5. → No (remains in FAULT): Go to Action A1.
A — ActionResolution
ActionsA1 (SAFE-SHUTDOWN persists after power cycle): Isolate machine at mains. Do not attempt to open machine. Mark out of service. Contact CoffCo field engineer immediately — SLA: 4 hours response. Log fault with time and machine serial number. A2 (fire/smoke/burning smell): Hotel emergency procedure. Mains isolation. Do not attempt to recover machine. A3 (fluid around base): Mains isolation. Do not restore power. CoffCo field engineer to attend and inspect electrical safety before machine is returned to service. NOTE: SAFE-SHUTDOWN recovery requires field engineer investigation. VS maintenance team must not attempt internal access.
FIDA Example 3 — Machine shows FLUID-EMPTY / no brew on command
F — Fault IdentificationSymptom / Entry Point
Observable symptomGuest or housekeeping reports: machine accepts selection but does not start brew. Status indicator shows EMPTY or LOW. Cloud ops alert FLUID-EMPTY received.
D — DiagnosisStructured Checks
Check sequenceD1: Is water reservoir empty or below MIN line? — Check reservoir. If empty/low → go to Action A1. D2: Is consumable correctly seated? — Open extraction chamber. Is consumable present and fully seated? If misloaded → go to Action A2. D3: Is consumable format correct? — Does the consumable appear to be a non-CoffCo item or wrong format? If yes → go to Action A3. D4: Reservoir full and consumable correct, but still FLUID-EMPTY → go to Action A4 (potential level sensor fault — escalate).
A — ActionResolution
ActionsA1 (empty reservoir): Refill reservoir to MAX. FLUID-EMPTY state clears automatically on refill. [Housekeeping / VS maintenance] A2 (misloaded consumable): Open chamber, remove and reseat consumable correctly until it clicks. Close chamber. Retry brew. [VS maintenance] A3 (wrong consumable format): Remove incorrect item. Insert correct CoffCo consumable. Retry brew. Note: extraction chamber interlock (SRS 1.4) will prevent chamber closing on wrong format — if guest has forced a non-CoffCo item, inspect chamber for damage before returning to service. [VS maintenance — visual inspection] A4 (level sensor suspected): Power cycle. If FLUID-EMPTY persists with full reservoir: escalate to CoffCo field engineer. Do not continue to operate — heating element inhibit is a safety function.
