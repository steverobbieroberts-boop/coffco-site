INITIALISING
Self-test · warm-up · hardness calibration
▼

Power-on self-test sequence. Firmware integrity check, sensor validation, hardness calibration, subsystem health check. Approximately 15–30 seconds.

Entry conditions
  • Power applied to machine
Exit transitions
  • Self-test OK → STANDBY
  • Self-test FAIL → FAULT
SRS / Safety refs: SRS 1.7, SRS 3.12
STANDBY
Idle · low power · monitoring active
▼

Machine idle, low-power mode. Telemetry heartbeat active. UI status indicator showing ready state. Awaiting user selection. Normal resting state between brews.

Entry conditions
  • Self-test OK from INITIALISING
  • Brew cycle complete from DISPENSING
  • Fault cleared from FAULT
  • OTA update complete from OTA-UPDATE
Exit transitions
  • User selection → BREWING
  • Fault detected → FAULT
  • OTA trigger (03:00–06:00) → OTA-UPDATE
  • Safety trigger → SAFE-SHUTDOWN
SRS / Safety refs: SRS 1.7, SRS 1.9
BREWING
Heating · pressurising · extracting
▼

Heating element energised. Water heating to 60–90°C. Pump building extraction pressure to 0.9–1.3 bar. Consumable loaded and sealed. Temperature and pressure monitored continuously.

Entry conditions
  • User selection confirmed (2-step CONFIRM BREW)
Exit transitions
  • Target temp/pressure achieved → DISPENSING
  • Fault detected → FAULT
  • Safety trigger → SAFE-SHUTDOWN
SRS / Safety refs: SRS 1.1, 1.2, 1.5, 3.10, 3.13
DISPENSING
Delivering beverage · measuring volume
▼

Beverage being delivered to guest mug. Flow sensor measuring volume. Target volume 200–400 ml in 50 ml steps. Dispensing completes when target volume reached.

Entry conditions
  • Target temperature and pressure achieved from BREWING
Exit transitions
  • Volume reached → STANDBY
  • Fault detected → FAULT
  • Safety trigger → SAFE-SHUTDOWN
SRS / Safety refs: SRS 1.3, 2.2, 2.7, 4.6, 4.7
LOW CONSUMABLE
Status indicator active · brewing still possible
▼

Consumable level below 20% threshold. Brewing is still possible. Status LED indicates low consumable state. Cloud ops alert transmitted. Machine continues to operate normally — this is a warning state only.

Entry conditions
  • Level sensor threshold reached during any operational state
Exit transitions
  • Housekeeping replenishes → previous operational state
  • Level exhausted → EMPTY/UNAVAILABLE
  • OTA trigger → OTA-UPDATE (STANDBY-equivalent)
SRS / Safety refs: SRS 1.4, 1.9
EMPTY UNAVAILABLE
No brewing possible · wireless alert sent
▼

Consumable or water store exhausted. No brewing possible. UI shows unavailable state. Wireless alert sent to cloud ops and housekeeping. Machine safe but non-functional until restocked.

Entry conditions
  • Consumable exhausted from LOW CONSUMABLE
  • Water reservoir empty
Exit transitions
  • Housekeeping replenishes → STANDBY
SRS / Safety refs: SRS 1.4, 1.9, SRS 1.10
FLUID EMPTY
Heating element inhibited · SRS 1.10 · SAF-02
▼

Water reservoir level below minimum threshold for a brew cycle. Heating element inhibited — dry-fire interlock active (SRS 1.10, FW-SF-09). Brew cycle aborted. EMPTY telemetry transmitted. Automatic recovery when reservoir refilled above minimum.

Entry conditions
  • Brew command received with level < minimum
Exit transitions
  • Refill above minimum (automatic) → STANDBY
SRS / Safety refs: SRS 1.10, SAF-02, FW-SF-09
OTA UPDATE
03:00–06:00 window · STANDBY only · A/B partition
▼

Firmware update in progress. Only entered from STANDBY, within 03:00–06:00 local time window, after 30-minute activity-free check. A/B partition install — automatic rollback to previous partition if STANDBY not achieved within 120 s post-reboot.

Entry conditions
  • OTA trigger from cloud ops (STANDBY state only, night window only)
Exit transitions
  • Update complete → STANDBY (UPDATE-COMPLETE telemetry)
  • Update failed → FAULT (UPDATE-FAILED, rollback to previous partition)
SRS / Safety refs: SRS 1.11, FW-SF-09
MAINTENANCE
Housekeeping service cycle · panel open
▼

Housekeeping service cycle active. Panel open for consumable restock or drip tray emptying. Brewing inhibited. Machine safe. No tools required for housekeeping tasks.

Entry conditions
  • Housekeeping opens access panel
Exit transitions
  • Panel closed / complete → STANDBY
SRS / Safety refs: SRS 1.4, 2.5, SRS 3.3
FAULT
Brewing suspended · wireless alert sent
▼

Recoverable fault detected — temperature or pressure out of range, sensor read failure, firmware watchdog reset. Brewing suspended. Wireless alert sent to cloud ops. Recovery possible after fault is cleared by field engineer or automatic reset.

Entry conditions
  • Temperature or pressure out of range
  • Sensor read failure
  • Firmware watchdog reset
  • OTA update failed (rollback)
Exit transitions
  • Fault cleared by service / reset → INITIALISING
SRS / Safety refs: SRS 1.7, 1.9, SRS 3.9, 3.12
SAFE SHUTDOWN
All actuators de-energised · recovery: power cycle only
▼

All actuators de-energised immediately. Entered from ANY state on safety trigger. Heating element, pump, frother, valves all off. Field engineer notified via cloud ops. Recovery by power cycle only — cannot be reset by software or housekeeping.

Entry conditions
  • SAF-01 (overtemperature TCO)
  • SAF-07 (insulation fault)
  • SAF-11 (interlock failure)
  • SAF-12 (watchdog timeout in safety function)
Exit transitions
  • Power cycle → INITIALISING (after field engineer clearance)
SRS / Safety refs: SAF-01, SAF-07, SAF-11, SAF-12
