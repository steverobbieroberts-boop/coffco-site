SRS requirement
SRS 1.11
Design intent. The 30–60 second machine reboot during a firmware update must never coincide with an active or imminent guest brew. Night-window scheduling eliminates this risk by restricting updates to the lowest-usage window (03:00–06:00 local) across all Vacation Stables properties. The state machine is the gating mechanism — update can only proceed from STANDBY.
🎯
1. Design Objectives
Six objectives derived from the SDA question: can firmware updates be scheduled during night hours to minimise guest impact?
Ref Objective Engineering requirement
OBJ-1Zero guest impactNo firmware update shall interrupt or prevent a guest brew cycle in progress or imminent.
OBJ-2Night-window deploymentAll firmware updates shall be scheduled for deployment in the 03:00–06:00 local time window of the property where each machine is installed.
OBJ-3Safe failureIf a firmware update fails for any reason, the machine shall revert to the previous firmware version and return to STANDBY state. Under no circumstances shall a failed update leave the machine in an unsafe state.
OBJ-4No user action requiredThe update process shall be entirely transparent to the hotel guest. No guest action, acknowledgement, or awareness is required.
OBJ-5Canary deploymentUpdates shall be deployed to a small canary group first, validated, then rolled out progressively across the fleet to limit exposure to any defect in the new firmware.
OBJ-6TraceabilityEvery machine's firmware version shall be recorded in the cloud platform asset register before and after update. Failed updates shall be logged with failure reason.
⚙️
2. Update Process — Four-Step Architecture
Cloud scheduler → machine check → download and verify → install and reboot
Step 1
Cloud Scheduler
Cloud ops schedules update release. Assigns firmware version, target device group, night window (local time per property time zone). Staggered canary release plan defined.
Step 2
Machine Firmware Check
Machine polls cloud at scheduled time. Checks: (a) state = STANDBY, (b) no recent brew activity (30-min activity-free check), (c) Wi-Fi signal adequate. All must pass.
Step 3
Download and Verify
Machine downloads firmware package. Verifies digital signature against CoffCo public key. Verifies checksum. If either fails: abort and log. Do NOT install.
Step 4
Install and Reboot
Verified firmware installed to inactive A/B partition. Machine reboots. Post-boot self-test. If STANDBY reached within 120s: confirm update. If not: automatic rollback to previous partition.
🔒
3. State Machine Gating
Update proceeds only from STANDBY — the FSM is the primary protection mechanism
STANDBY
✓ Proceed (all checks pass)
Machine is idle. No guest activity. Safe to update.
LOW CONSUMABLE
✓ Proceed (STANDBY-equivalent)
Low consumable is a status indicator only — machine is otherwise operational and idle.
BREWING
⏸ Defer — re-check every 5 min
Update must never interrupt an active brew cycle. Heating element may be energised.
DISPENSING
⏸ Defer — re-check every 5 min
Beverage is being delivered to guest. Do not interrupt.
MAINTENANCE
⏸ Defer until STANDBY
Housekeeping is servicing the machine. Do not interrupt.
FLUID-EMPTY
⏸ Defer until STANDBY
Machine is in a degraded state. Update when restored to normal operation.
SAFE-SHUTDOWN
✗ Block — alert cloud ops
Safety state. Machine may have a hardware or firmware fault. Update only after Tier 2 engineer has cleared the fault.
FAULT
✗ Block — alert cloud ops
Fault state. Root cause unknown. Update would mask or complicate diagnosis.
⏱️
4. Activity-Free Check
State check alone is insufficient — a guest may be about to start a brew

Even in STANDBY state, a guest may have selected a drink type but not yet pressed CONFIRM BREW. A 30-minute activity buffer provides a conservative gate against this scenario.

ConditionAction
Last UI activity >30 min agoActivity-free check passes — proceed with update.
Last UI activity within 30 min of 03:00Update deferred by 30 minutes and re-checked. Maximum deferral: until 05:30 local time (leaving 30 minutes of update window).
Continuously in STANDBY since 22:00 localActivity-free check waived — machine is clearly unoccupied. Proceed immediately.
